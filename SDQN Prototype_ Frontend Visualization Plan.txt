Project Overview: SDQN Prototype Documentation


Objective: To build a prototype Software-Defined Quantum Network (SDQN) that mitigates service blocking caused by key exhaustion in Quantum Key Distribution (QKD) networks. The system utilizes a novel Quality-of-Security (QoSec) routing algorithm that treats secret keys as a dynamic resource.


1. System Requirements & Toolchain
To implement the hybrid architecture, the following stack is required:
* Core Language: Python 3.9.

* Classical Control Plane (Routing): NetworkX for graph-based routing calculations.

* Quantum Data Plane (Network Physics): QuNetSim for discrete-event simulation of quantum nodes and channels.

* Quantum Data Plane (Sub-atomic Physics): IBM Qiskit (custom integration) to simulate actual qubit generation, superposition, and measurement.
* Frontend Dashboard: Streamlit or Dash (Python-based frameworks) to build the real-time interactive visualization.
2. Current Implementation Status (What's Been Done)
The logical framework of the paper has been successfully implemented and verified:
   * Network Construction: The SDN_Controller class successfully initializes a global knowledge map with a 10-node, 18-link mesh topology representing a metropolitan area network.

   * QoSec Routing Engine: The custom link cost function has been successfully mapped to NetworkX shortest-path algorithms, pruning links when the key buffer drops below 50 bits. The formula implemented is:

$$C_{link}(u, v) = \alpha(D_{uv} / D_{max}) + \beta(K_{threshold} / K_{curr}(u, v)) + \gamma e^{(\eta \cdot QBER_{uv})}$$
   * Simulation Environment: User traffic is correctly generating key requests via a Poisson distribution ($\lambda=2$).

   * Adversary Injection: The "Eve" thread is actively intercepting and resending data every 100 ticks, successfully causing the Quantum Bit Error Rate (QBER) to spike to 15%.

3. Immediate Next Steps (The "Physics" Engine)
Before building the visual layer, the data plane stubs must be replaced with actual physics simulations.
      * Integrate Qiskit with QuNetSim: Replace the background key generation stub with a function that calls Qiskit to generate raw bits using the BB84 protocol at a base rate of 10 kbps.

      * Calculate Live QBER: Program Qiskit to compare Alice's and Bob's sifted keys to mathematically generate the QBER, rather than relying on a hardcoded variable.
